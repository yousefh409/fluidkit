# Changelog

## Unreleased

- **LiquidCard.** The base content surface: children live in normal flow (the surface is a backdrop layer BEHIND the content, never a clipped parent of it — text can never scale by construction) and a ResizeObserver rebuilds the engine geometry to follow. Callout `variant` presets (info/success/warning) tint through the glass, no fake gradients. Static — no animation loop, no reduced-motion branch needed.
- **Shared surface lighting.** New `intensity` scale for the surface family (`0`–`1`, with `"whisper"`/`"present"` presets, `src/components/intensity.ts`) and a shared rim-light module (`src/components/rim.ts`): a crisp 1px gradient ring around the whole border, brightest toward the scene light, plus an inset glow whose blur scales with surface height. CSS mask-ring + inset shadow — safe only on static rounded rects, where the ring and the engine clip path can never disagree.
- **MeniscusDivider.** A divider rule as a bead of liquid: a thin engine pill spanning its container, lifted by shadow, glinted and rim-lit on glass (`role="separator"`). Mercury/flat stay unlit per the material rules.
- **LiquidPanel.** Drawer/sheet surface that pours in from one edge (`side`): anchored engine geometry grows across the box on a spring, corner radius collapsing as the sliver thins; content rises in on the unclipped layer only after the surface arrives. Reduced motion snaps the geometry; the content cross-fade is the only motion left.
- **LiquidTooltip.** A tooltip as a droplet: the label pill condenses beside the trigger on a spring (`speed` multiplier scales the pace without changing the motion's character), with a tail bead in deep overlap — fused through the engine's real tension bridges, so the meniscus neck is permanent. Label text defines the droplet's size and lives above the engine subtree. `role="tooltip"` + `aria-describedby`, focus shows without delay, Escape dismisses; hover delay defaults to 100 ms. Plain CSS positioning (no portal); `overflow: hidden` ancestors will crop it.
- **LiquidText.** Glyphs made of glass: a backdrop-blurred layer masked by an SVG rendering of the same string (same live font styles) paints translucent letterforms over the page, while the real text stays in flow as invisible ink — layout, selection, and screen-reader semantics intact. A narrow specular sheen sweeps across on a zero-JS `@keyframes` loop (parks on the light-facing third under reduced motion; pauses in-phase off-screen). `material="ink"` keeps solid-color glyphs (also the fallback for non-string children and missing backdrop-filter; SVG masks can only use system fonts, so exotic web-font pages should prefer ink).
- **LiquidDialog.** A modal that morphs from its trigger: at open it captures the focused element's rect (or an explicit `origin`) and the surface springs from that button-sized pill up to the centered dialog, expanding with one taut overshoot — translate + geometry only, content fades in after arrival. The backdrop is shallow water (tint + backdrop blur that itself animates, since `backdrop-filter` ignores element opacity). Close drops it back toward the trigger, the fade waiting until it has mostly landed, the portal unmounting only after both finish. Escape/backdrop-click close, body scroll locks, focus moves in and restores. Portal to `document.body`.
- **VoiceBall.** A voice-assistant orb driven by a live `level` prop (0–1, smoothed internally — feed it from any audio stack at 60fps, it lands in a ref) and a `mode`: `idle` (slow ambient breathe, level ignored), `listening` (quicker breathe + half-gain level response), `speaking` (full swell, fast edge undulation on two sine harmonics, satellite beads surfacing at peaks and fusing back through real tension bridges). Deterministic phases, off-screen pause, reduced motion renders a static circle.
- **Showcase pages for the surface family (site only).** Seven new registry pages — LiquidCard, MeniscusDivider, LiquidPanel, LiquidTooltip, LiquidText, LiquidDialog, VoiceBall — each with live controls; VoiceBall's hero is driven by a deterministic fake speech envelope (no mic, no randomness).
- **Bundle-size budget re-pinned: 26.1 kB brotli** (from 14.9 kB, and from the backgrounds branch's 18.2 kB). With the seven surface components merged alongside the FlowStagger/backgrounds refreshes, the core entry measures 21.77 kB; re-pinned at the usual +20% headroom.
- **FlowStagger liquid pass.** Entrance is now a viscous cascade (gaps grow ~5% per item, capped) landing on a soft spring with slight overshoot; default `stagger` drops `0.06` -> `0.02`. Sibling FLIP glides ripple outward from the change point (~24ms per position, capped at 150ms) via a child-key diff against the last committed list. Removals submerge — sink + blur-out + slight shrink — while the row's height collapses on an ease-out, so the space closes over it; the container carries `layout` (inside a `LayoutGroup`) so any background it wears breathes with the list instead of snapping. Reduced motion still collapses everything to a simultaneous fade. Breaking (pre-1.0): `useFlow` now returns `getItemProps({ entranceRank, glideDistance })` instead of a static `itemProps`, and container orchestration no longer uses `staggerChildren`. Showcase page rebuilt to match: glass rows over a gradient pool, Shuffle + click-row-to-top, capped lists in fixed-height reserves so nothing above a change ever shifts.

- **Component showcase (site only, nothing shipped in the package).** The playground is now a sidebar-navigated showcase: one page per primitive (hero stage with live controls, a fixed-prop variants grid, a usage snippet) plus a Demos gallery of the four phone-frame recipes. A tiny hand-rolled hash router mounts only the active page, so GPU pages boot WebGL only when visited; shared page chrome lives in `playground/showcase/kit/`, pages register via one line each in `playground/showcase/registry.ts`. `playground/tsconfig.json` typechecks the site with the vite aliases mapped. Fixed in passing: the old Ripple card called 650 ms the default when the component default is 600 ms.

- **LiquidMetal.** Optional GPU tier: a real WebGL liquid-metal shader wrapping `@paper-design/shaders-react` (pinned exact `0.0.76`, since the package ships breaking changes across `0.0.x` releases), behind the `fluidkit/liquid-metal` subpath export. A raw passthrough: `LiquidMetalProps` extends the shader's own param surface (colors, stripe pattern, dispersion, shape/image mask, sizing, speed) with every knob as a top-level prop; defaults mirror the upstream full-screen "Backdrop" preset (`shape: "none"`, `scale: 1`), so it fills its container with flowing metal instead of the upstream default's small floating diamond (a pin-bump canary test keeps the defaults in sync with the real package). No WebGL or reduced motion falls back to a static metallic-gradient frame, the shader never mounts; off-screen the shader stays mounted but its speed is forced to `0`, which stops its render loop entirely — the pause gate always wins over any prop.
- **GPU tier packaging.** The GPU primitive ships as an isolated `tsup` entry and optional peer dependency (`peerDependenciesMeta.optional`): installing fluidkit without it never warns, and importing the core `fluidkit` entry never resolves the GPU library. `check:gpu-leak` now also scans the shared chunk files a core entry imports (not just the entry file itself), so bytes leaking into a shared chunk are caught, not just bytes in the entry point directly. `check:pack` verifies the 6 new dist artifacts (`liquid-metal` x `js`/`js.map`/`cjs`/`cjs.map`/`d.ts`/`d.cts`) ship in the published package. (A `WaterField` fluid-sim primitive wrapping `webgl-fluid-enhanced` existed briefly during development and was removed before release.)
- **MeshGradient.** Ambient CSS backdrop: 3-4 large, softly blurred radial-gradient blobs drift on long-period (`speed`-scaled) `@keyframes` transform loops, zero per-frame JS once mounted. Blob placement and phase are deterministic (golden-angle spread, no `Math.random`). Reduced motion drops the keyframes entirely; off-screen pauses `animation-play-state` instead of tearing the loop down.
- **GlassPanes.** Glass-native ambient backdrop: frosted panes on a shared slight diagonal, sliding past each other; each pane backdrop-blurs at a different strength so overlaps read as physical depth. Panes tile the FULL surface — widths and slot spacing are derived from `count` with overlap margins that absorb the drift, jitter, and shared rotation, so every point of the container is always behind at least one pane. `colors` tints fold into the glass white scaled by `intensity`; degrades to layered frosted fills (`data-fallback`) without `backdrop-filter`; same reduced-motion and off-screen rules.
- **Silk.** Ambient CSS backdrop: full-height diagonal gradient sheets drift and breathe like slow-motion fabric, all on a single shared flow direction (one composition, same reasoning as MeshGradient's single light direction). `count` (1-12) scales density independently of the palette (sheets cycle `colors`); `material="glass"` renders frosted, backdrop-blurring white-tinted sheets via the engine's material resolver, degrading to the plain tint without `backdrop-filter` support. Deterministic golden-ratio layout; same reduced-motion and off-screen rules.
- **JellyButton.** An engine pill button that squashes on press via geometry, not a CSS transform, so the label never scales. Real `<button>` semantics (focus, Enter/Space, `disabled`); volume-preserving squash with spring-overshoot jiggle on release; material/tint/light/reflection/refraction passthrough. The press is point-aware by default (`deformPress`): the outline dents around the pointer and the displaced mass bulges away from it (keyboard presses stay symmetric), with an expanding specular glint from the press point on glass (`pressGlint`; off when `reflection={false}` or `light={null}`) and a pressed fill deepening with an asymmetric fade — quick in, slow out (`pressFeedback`, custom color via `pressColor`). `releaseWave` opts into a decaying outline ripple on release; `spring` overrides the feel, and the settle window is derived from the spring rather than a fixed timeout. Reduced motion keeps a fully functional button with the opacity press dip plus the fill deepening (color, not motion).
- **useSquish.** Headless press-squash hook: volume-preserving `scaleX`/`scaleY` Motion values plus pointer/keyboard handlers, for jelly-pressing arbitrary elements via CSS transform. Reduced motion makes pressing inert. Default spring shared with `JellyButton`, tuned to one soft overshoot (`stiffness 550 / damping 30`) — restraint over wobble.
- **Magnetic.** Behavior wrapper that pulls its child toward the pointer within a `radius` (linear falloff, travel hard-capped at `radius / 2`) and springs back outside it. Window-level tracking attaches only while on screen; reduced motion attaches nothing.
- **LiquidDrag.** Motion's own drag gesture plus a velocity-driven, volume-preserving stretch (continuous axis split, no cliff at the diagonal) that wobbles back to rest on release. `elasticity={0}` and reduced motion pin the scales at exactly 1 while dragging stays functional.
- **DripFuse.** One-shot liquid transfer: a drop swells off a source body, tears free via the engine's tension hysteresis, flies to a target, and fuses in while the target swells to absorb it. Fired by a `fire` counter; rapid fires coalesce into one `onComplete`. Reduced motion completes instantly.

## 0.3.0

- **LiquidTabs on the liquid engine.** The indicator is now an engine body: on tab change the old pill drains while the new one fills, a metaball tension bridge stretches between them, snaps free past the snap distance, and the new pill settles on a taut spring. Labels stay on an unfiltered sibling layer; reduced motion snaps instantly.
- **Pointer interaction API.** `Droplets` gains `interactive` plus `onGrab` / `onTear` / `onRelease`: grab a drop, drag it out (the neck stretches), tear it off past the snap distance, and it springs back and re-merges on release. The physics are the engine's normal tension hysteresis.
- **Performance pass.** `LiquidRenderer` exposes an imperative `LiquidSceneHandle`; `Droplets`, `MorphSurface`, and the tabs indicator write per-frame clip-path strings and specular ellipse attributes straight to the DOM — animation loops no longer commit React updates.
- **Morph polish.** Satellites ride their own softer spring (`stiffness 150 / damping 14`) instead of sharing the body spring; `useMotionSprings` supports per-slot configs and single-slot retargeting.
- **Opt-in refraction.** `refraction?: boolean` (default off) on `Droplets`, `Thinking`, and `MorphSurface`: Chromium-only edge lensing via an SVG `feDisplacementMap` inside `backdrop-filter`, gated on feature detection; degrades silently to plain glass blur.
- **Goo stack deleted.** `useGoo` and `src/filters/*` are gone — nothing uses the blur+contrast filter anymore.
- **Docs site.** The playground is now the public site: hero, per-component sections with live demos, controls, and copy-paste snippets; `npm run build:site` produces a deployable static bundle.

## 0.2.0

- **The liquid engine.** Real metaball geometry (circle / rounded-rect subpaths plus bezier bridge curves) applied per frame as a live `clip-path`; spring-driven motion via Motion; surface tension with hysteresis (connect on touch, stretch, snap past ~1.3x combined radii; the neck never thins to a hairline).
- **Materials as a prop.** The same shape renders as `glass` (white tint + backdrop blur/saturation, specular highlights from one shared, configurable scene light, toggleable via `reflection`), `mercury` (solid liquid metal — no gradient, no painted highlight), or `flat` (plain color; automatic fallback when `backdrop-filter` is unsupported).
- **Primitives rebuilt on the engine.** `Droplets` (tension cluster + optional pointer-chasing drop), `Thinking` (merge/split status indicator), `MorphSurface` (pill → panel with satellite absorption; content faces only cross-fade — text never scales, never rasterizes).
- **Rendering constraints encoded as tests.** clip-path and backdrop-filter never share an element; no CSS blur() for decorative glows; inline svg overlays carry explicit sizes; glass never backdrop-samples its own shadow.
- Light-mode-first playground rebuilt around the engine.
