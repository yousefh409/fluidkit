# GPU Tier Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `LiquidMetal` (wrapping `@paper-design/shaders-react`) and `WaterField` (wrapping `webgl-fluid-enhanced`) behind isolated subpath exports with optional peer dependencies, adding the fluidkit contract the raw libraries lack.

**Architecture:** Two new tsup entries (`src/liquid-metal/index.tsx`, `src/water-field/index.tsx`) building to `dist/liquid-metal.*` and `dist/water-field.*`, exposed as `fluidkit/liquid-metal` and `fluidkit/water-field` in the exports map. The wrapped libraries are optional peers (plus devDependencies for local dev): importing the core entry never resolves them (CI leak check), installing fluidkit without them never warns (`peerDependenciesMeta`). Each wrapper adds: WebGL feature detection with graceful fallback, `prefers-reduced-motion` collapse to a static frame, off-screen pausing, SSR safety, and fluidkit-consistent prop names.

**Tech Stack:** `@paper-design/shaders-react` (pin EXACT `0.0.76` â€” upstream ships breaking changes under 0.0.x), `webgl-fluid-enhanced` (`^0.8.0`, MIT, stable API), tsup multi-entry, Vitest with mocked GPU deps (jsdom has no WebGL).

**Sub-project 4 of 5** in `docs/superpowers/specs/2026-07-01-fluidkit-v1-design.md`.

**Canonical patterns:** wrapper conventions and gating from `src/components/MeshGradient.tsx` / `Aurora.tsx` (aria-hidden decoration, data-fluidkit, data-animating, useInView, usePrefersReducedMotion, resolveColor); feature detection style from `src/utils/featureDetect.ts` (lazy functions, error-swallowing, SSR-safe false).

**Hard constraints:** zero bytes from either GPU package in the core bundle (`check:gpu-leak` â€” its `distFiles` list must NOT gain the new bundles); `check:pack` `requiredFiles` MUST gain the new dist artifacts; core `size-limit` budget (14.9 kB) must not move (subpaths are separate entries and unbudgeted for now); reduced motion must never boot a WebGL sim; all detection lazy and SSR-safe; fallbacks are fluidkit-quality, not blank divs.
