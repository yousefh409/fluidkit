# fluidkit v0.3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship fluidkit v0.3 in one PR: LiquidTabs rebuilt on the liquid engine (bridge-flow indicator), a pointer grab/tear interaction API on Droplets, a per-frame-DOM-write performance pass, softer satellite springs, opt-in Chromium refraction, publish metadata, and the playground polished into the public docs site.

**Architecture:** All animated shapes go through the existing liquid engine (`src/liquid`): metaball subpaths → one `clip-path` string. The perf pass gives `LiquidRenderer` an imperative handle so rAF loops write clip-path/ellipse attrs straight to DOM refs instead of `setState`. LiquidTabs models the tab change as mass draining from the old tab pill into the new one through a tension bridge that snaps when stretched. Refraction is an opt-in per-instance SVG displacement filter referenced from `backdrop-filter`, gated on `supportsRefraction()`.

**Tech Stack:** React 18, Motion (springs), vitest + Testing Library (jsdom, `vi.doMock("motion/react")` pattern), tsup, vite playground.

**Ordering rationale:** engine plumbing first (springs config → renderer handle), then component migrations that use it, then the tabs rebuild, then goo deletion (tabs was the last goo doc reference), then new features (interaction, refraction), then release chores (metadata, site, PR).

### Task 2: Imperative scene handle on `LiquidRenderer`

**Files:**
- Modify: `src/liquid/LiquidRenderer.tsx`, `src/liquid/index.ts`

**Design:**

```ts
export interface LiquidScene {
  path: string;
  speculars?: SpecularSpot[];
}
export interface LiquidSceneHandle {
  /** Write the scene straight to the DOM (no React render). */
  setScene(scene: LiquidScene): void;
}
```

- `LiquidRenderer` becomes `forwardRef<LiquidSceneHandle, LiquidRendererProps>`.

- Declarative render still works exactly as before (SSR/static path).

### Task 5: LiquidTabs on the liquid engine (bridge-flow indicator)

**Decided behavior:** on tab change the indicator (an engine body) stretches a tension bridge toward the target tab; mass drains from the old pill into the new one; when the bridge's stretch passes `SNAP_STRETCH` it snaps free; the new pill settles on a taut spring. Labels stay on an unfiltered sibling layer. Reduced motion: instant snap.

**Files:**
- Rewrite: `src/components/LiquidTabs.tsx`
- Modify: `tests/components/LiquidTabs.test.tsx` (only where the DOM mechanism changed)
- Modify: `docs/primitives/liquid-tabs.md`

**Model (per frame during a transition):**

```
drain: spring 1 → 0   (old pill's remaining mass)   TABS_DRAIN_SPRING { stiffness: 260, damping: 26 }
fill:  spring 0 → 1   (new pill's arrived mass)     TABS_FILL_SPRING  { stiffness: 420, damping: 30 } (taut, slight overshoot)

fromRect, toRect = measured offsetLeft/offsetWidth of prev/next tab; h = container height
path  = roundRect(fromCenter, fromRect.width * drain, h * clamp(drain * 3, 0, 1), h/2)
      + roundRect(toCenter,   toRect.width  * fill,  h * clamp(fill * 3, 0, 1),  h/2)
      + bridge while connected:
          bodyFrom = { x: fromCx, y: cy, r: (h / 2) * clamp(drain * 1.4) }
          bodyTo   = { x: toCx,   y: cy, r: (h / 2) * clamp(fill * 1.4) }
          stretch  = dist / (rFrom + rTo)   // grows as the old pill drains → natural snap
          connected starts true at flip; set false permanently once stretch > SNAP_STRETCH or either r < 0.5
          path += bridgePath(bodyFrom, rFrom, bodyTo, rTo, neckWaist(min(stretch, SNAP_STRETCH)))
```

At rest (no transition) and under reduced motion: `path = roundRect(activeCenter, activeWidth, h, h/2)` rendered declaratively; no loop. The rAF loop only runs while `settling` (same SETTLE_MS window pattern as MorphSurface, ~1100ms).

**Rendering:** indicator layer keeps `data-fluidkit="liquid-tab-indicator-layer"` (absolute, inset 0, pointer-events none, aria-hidden). Inside it, a div `data-fluidkit="liquid-tab-indicator"` (absolute, inset 0) containing `<LiquidRenderer ref path material={resolveMaterial("flat", { color: resolvedColor })} />`. No shadow, no speculars. The indicator color therefore lands on `[data-fluidkit="liquid-fill"]`'s `background`.

**Public API unchanged:** `items`, `value`, `onChange`, `color`, container `role="tablist"`, `data-motion` attr.

### Task 7: Pointer interaction API on Droplets (grab / drag / tear / re-merge)

**Decided API (documented in `docs/primitives/droplets.md`):**

```ts
/** Drops can be grabbed with the pointer, dragged (neck stretches), torn off
 * past the snap distance, and spring home / re-merge on release. */
interactive?: boolean;              // default false
onGrab?: (index: number) => void;   // pointer picked up a drop
onTear?: (index: number) => void;   // its last bridge snapped while dragging
onRelease?: (index: number) => void; // pointer let go (drop springs home)
```

Under reduced motion `interactive` is inert (static dots, no handlers needed).

**Files:**
- Modify: `src/components/Droplets.tsx`, `src/liquid/tension.ts`, `src/liquid/index.ts`
- Test: `tests/components/Droplets.test.tsx`, `tests/liquid/tension.test.ts`
- Modify: `docs/primitives/droplets.md`

**Mechanics:**
- `TensionField` gains a query: `connectedTo(id: string): boolean` — whether any live bridge involves `id`. (Unit-test in `tension.test.ts`: connect two bodies, expect true; separate past snap, expect false.)
- `onPointerDown` (only when `interactive && animating`): hit-test bodies (`dist(pointer, body) <= body.r * 1.25`); on hit → `setPointerCapture`, store `grabbed = { index, wasConnected }`, set `data-grabbed={index}` on the root, fire `onGrab`.
- `onPointerMove` while grabbed: `springs.setTargets` for that drop's x/y slots to the pointer position with `GRAB_SPRING = { stiffness: 550, damping: 38 }` (tight liquid lag). Other slots keep their current targets (pass their home targets).
- Each rAF frame while grabbed: if it *was* bridge-connected and `tension.connectedTo(id)` turned false → fire `onTear` once (the hysteresis in TensionField provides the actual stretch/snap physics for free).
- `onPointerUp` / `onPointerCancel`: clear grab, `setTargets` back to the squeeze-phase home targets with `DROP_SPRING`, fire `onRelease`. Re-merge on arrival happens naturally via `CONNECT_STRETCH`.
- `followPointer` unchanged; while a grab is active the chase drop is ignored (guard `pointerActive` handling).

### Task 8: Opt-in refraction (`refraction?: boolean`, default OFF)

Chromium-only edge lensing via an SVG `feDisplacementMap` referenced from `backdrop-filter` (kube.io liquid-glass technique), gated on `supportsRefraction()`. Degrades silently to plain glass blur. Applies to the glass material on `Droplets`, `Thinking` (inherits via DropletsProps), and `MorphSurface`.

**Files:**
- Create: `src/liquid/refraction.tsx`
- Modify: `src/liquid/materials.ts`, `src/liquid/index.ts`, `src/components/Droplets.tsx`, `src/components/MorphSurface.tsx`
- Test: `tests/liquid/refraction.test.tsx`, `tests/liquid/materials.test.ts` additions
- Modify: `docs/primitives/droplets.md`, `docs/primitives/morph-surface.md`, `docs/primitives/thinking.md`

**Design (`src/liquid/refraction.tsx`):**

```tsx
/** Pure: data-URI SVG displacement map. Red = X displacement, green = Y,
 * neutral gray center (no displacement) fading to full displacement at the
 * rim — so only edges lens, like thick glass. */
export function displacementMapUri(width: number, height: number): string;

export interface RefractionDef {
  /** `url(#id)` to prepend to the glass backdrop-filter, or null (unsupported/off). */
  url: string | null;
  /** Hidden inline defs svg (explicit width/height 0) to render, or null. */
  defs: ReactNode;
}

/** SSR-safe: detection runs in render via supportsRefraction() (guarded, never throws). */
export function useRefraction(enabled: boolean, width: number, height: number): RefractionDef;
```

The defs svg: `<svg width="0" height="0" aria-hidden style={{position:"absolute"}}>` containing `<filter id colorInterpolationFilters="sRGB" x="0" y="0" width="100%" height="100%"><feImage href={displacementMapUri(w,h)} result="map" ... /><feDisplacementMap in="SourceGraphic" in2="map" scale="28" xChannelSelector="R" yChannelSelector="G" /></filter>`. The map itself: full-canvas black base, a horizontal red 255→0 gradient rect and a vertical green 255→0 gradient rect composited with `screen`, then a centered radial overlay of neutral `rgb(128,128,0)`-equivalent gray fading out toward the rim.

**materials.ts:** `ResolveMaterialOptions` gains `refractionUrl?: string | null`; when glass + supported, `backdropFilter` becomes `` `${refractionUrl} blur(16px) saturate(1.8)` ``. clip-path stays on the wrapper — the constraint holds because only the *fill child* carries backdrop-filter.

**Components:** `refraction?: boolean` (default false) → `const { url, defs } = useRefraction(refraction && material === "glass", side, side);` pass `refractionUrl: url` into `resolveMaterial`, render `{defs}` inside the root container.
