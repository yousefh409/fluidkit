# fluidkit v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Plan style note:** Per project convention this plan is high-level — goals, steps, decisions, and test intent, not code. The engineer writes the actual code and tests during execution, following TDD (write the failing test first). Reference `../specs/2026-07-01-fluidkit-design.md` for the API contract and `../../../prototypes/` for the visual target.

**Goal:** Ship `fluidkit` v0.1 — a React library of fluid UI animations (morph-without-text-warp, goo metaballs, thinking blob, liquid glass, flow stagger, liquid tabs, ripple) built on Motion, tree-shakeable, SSR-safe, reduced-motion-aware.

**Architecture:** Single TypeScript package built with `tsup` (ESM + CJS + types). Shared infrastructure (auto-injected SVG filter defs, feature detection, reduced-motion, color, in-view pause) underpins each primitive. Components wrap Motion for all spring/layout/FLIP; the library adds only the fluid *material and organic motion* layer. LiquidGlass wraps `@samasante/liquid-glass` for real refraction with a frosted-blur degradation fallback.

**Tech Stack:** React 18+, Motion (peer), TypeScript, tsup, Vitest + Testing Library, `@samasante/liquid-glass` (dep), Vite playground, Playwright for visual verification.

## File structure (created across the plan)

```
package.json · tsconfig.json · tsup.config.ts · vitest.config.ts · vitest.setup.ts
src/
  index.ts                      re-exports every public component + hook
  filters/
    defs.tsx                    goo + lens <filter> markup (data, no side effects)
    injectDefs.ts               idempotent, ref-counted, SSR-guarded singleton injector
    useFilterDefs.ts            hook wrapping injectDefs for components
  utils/
    featureDetect.ts            backdrop-filter / refraction / view-transition detection
    reducedMotion.ts            resolve effective reduced-motion state
    color.ts                    normalize color input, currentColor default
    useInView.ts                IntersectionObserver pause hook
  hooks/
    useMorph.ts · useGoo.ts · useFlow.ts · useRipple.ts
  components/
    MorphSurface.tsx · Metaballs.tsx · ThinkingBlob.tsx · LiquidGlass.tsx
    FlowStagger.tsx · LiquidTabs.tsx · Ripple.tsx
tests/                          mirrors src/ (Vitest + Testing Library, jsdom)
playground/                     Vite app: one route per primitive, mirrors prototypes
docs/primitives/                one markdown doc per primitive
```

Decision: split by responsibility. Shared logic (`utils/`, `filters/`) is isolated and unit-tested so every primitive reuses one tested implementation of degradation, reduced-motion, and defs injection.

## Phase 1 — Shared infrastructure (TDD)

Rule for every task in this phase: write the failing test first, run it to confirm it fails, implement the minimum to pass, run to confirm green, commit.

### Task 1.1: Feature detection (`utils/featureDetect.ts`)

**Files:** Create `src/utils/featureDetect.ts`; Test `tests/utils/featureDetect.test.ts`.

- [ ] **Step 1:** Test that each detector (`supportsBackdropFilter`, `supportsRefraction`, `supportsViewTransition`) returns `false` (never throws) when the relevant API/`CSS.supports` is absent — the SSR/unsupported path.
- [ ] **Step 2:** Test that each returns `true` when the capability is present (mock `CSS.supports` / `document.startViewTransition`).
- [ ] **Step 3:** Implement pure guarded detectors; no work at module import. Commit.

### Task 1.2: Reduced motion (`utils/reducedMotion.ts`)

**Files:** Create `src/utils/reducedMotion.ts`; Test `tests/utils/reducedMotion.test.ts`.

- [ ] **Step 1:** Test a pure resolver that maps the reduce/no-preference matchMedia state to a boolean, defaulting safe (no window → treat as reduced-friendly = static-safe default documented in the test).
- [ ] **Step 2:** Implement the resolver plus a thin hook that reads Motion's `useReducedMotion` and falls back to the resolver. Commit.

### Task 1.3: Color normalization (`utils/color.ts`)

**Files:** Create `src/utils/color.ts`; Test `tests/utils/color.test.ts`.

- [ ] **Step 1:** Test that a provided color passes through, an omitted color yields `currentColor`, and CSS custom-property references are preserved.
- [ ] **Step 2:** Implement the normalizer. Commit.

### Task 1.4: In-view pause hook (`utils/useInView.ts`)

**Files:** Create `src/utils/useInView.ts`; Test `tests/utils/useInView.test.tsx`.

- [ ] **Step 1:** Test that the hook reports `false` before observation and flips to `true` when a mocked IntersectionObserver fires an intersecting entry; test it no-ops safely without IntersectionObserver (SSR/old browsers).
- [ ] **Step 2:** Implement with cleanup on unmount. Commit.

### Task 1.5: Filter defs data (`filters/defs.tsx`)

**Files:** Create `src/filters/defs.tsx`; Test `tests/filters/defs.test.tsx`.

- [ ] **Step 1:** Test that rendering the defs produces one `goo` filter (blur + color-matrix alpha contrast) and one `lens` filter (turbulence + displacement) with the library-namespaced ids used by consumers.
- [ ] **Step 2:** Implement as pure markup mirroring the prototype filter values (`prototypes/05` goo, `prototypes/02` lens). Commit.

### Task 1.6: Singleton defs injector (`filters/injectDefs.ts` + `useFilterDefs.ts`)

**Files:** Create `src/filters/injectDefs.ts`, `src/filters/useFilterDefs.ts`; Test `tests/filters/injectDefs.test.tsx`.

- [ ] **Step 1:** Test idempotency — calling inject twice mounts exactly one hidden `<svg>` defs node in the DOM.
- [ ] **Step 2:** Test ref-counting — two consumers mounting then one unmounting keeps the node; the last unmount removes it.
- [ ] **Step 3:** Test SSR safety — the injector performs no DOM work at import and is a no-op when `document` is undefined; the `useFilterDefs` hook injects only inside an effect.
- [ ] **Step 4:** Implement injector + hook. Commit.
